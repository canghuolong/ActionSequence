## 版本说明

### 版本号格式

ActionSequence 遵循 [语义化版本](https://semver.org/lang/zh-CN/) 规范：

```
主版本号.次版本号.修订号
```

- **主版本号**: 不兼容的 API 更改
- **次版本号**: 向后兼容的功能新增
- **修订号**: 向后兼容的问题修正

### 文档版本策略

1. **主版本**: 为每个主版本维护独立的文档分支
2. **次版本**: 在当前文档中更新，保留历史记录
3. **修订号**: 直接更新当前文档

## 维护旧版本文档

### 创建版本分支

当发布新的主版本时：

```bash
# 创建版本分支
git checkout -b docs/v1.0.0

# 在主分支更新到新版本
git checkout main
# 更新文档...
```

### 访问旧版本文档

旧版本文档可以通过以下方式访问：

1. **Git 分支**: 切换到对应的文档分支
2. **GitHub**: 在仓库中选择对应的分支
3. **发布包**: 下载对应版本的包

## 文档更新流程

### 修订号更新 (1.0.0 → 1.0.1)

1. 直接在当前文档中修复错误
2. 更新 CHANGELOG.md
3. 提交更改

### 次版本更新 (1.0.0 → 1.1.0)

1. 在当前文档中添加新功能说明
2. 更新 API 参考
3. 添加新示例
4. 更新 CHANGELOG.md
5. 更新迁移指南（如需要）

### 主版本更新 (1.0.0 → 2.0.0)

1. 创建旧版本文档分支
2. 在主分支更新文档
3. 编写详细的迁移指南
4. 说明破坏性更改
5. 更新所有相关文档

## 版本兼容性

### API 兼容性

- **主版本**: 可能包含破坏性更改
- **次版本**: 向后兼容
- **修订号**: 完全兼容

### 文档兼容性

- 每个版本的文档对应该版本的 API
- 旧版本文档保持不变
- 新版本文档包含迁移指南

## 最佳实践

1. **版本标记**: 在文档中明确标注版本信息
2. **更新日志**: 详细记录每个版本的更改
3. **迁移指南**: 提供清晰的升级路径
4. **示例更新**: 确保示例代码与版本匹配
5. **链接维护**: 保持文档间链接的正确性

## 文档归档

### 归档策略

- 主版本文档永久保留
- 次版本文档保留最近 3 个版本
- 修订号文档仅保留最新版本

### 归档位置

- **Git 分支**: `docs/v{major}.{minor}.{patch}`
- **发布包**: 包含对应版本的文档
- **网站**: 提供版本选择器
